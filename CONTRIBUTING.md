# Contributing to Research Cycle Skill

Thank you for your interest in contributing!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Include clear reproduction steps for bugs
- Tag issues appropriately (bug, enhancement, question)

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test the skill in Codex to verify it works
5. Submit a pull request with a clear description

### Skill Design Guidelines

When modifying the SKILL.md:

- **Keep it general**: This skill should work for any research domain, not just specific fields
- **Preserve structure**: Maintain the 7-phase workflow structure
- **Add templates**: New phases or sub-phases should include markdown templates
- **Document decisions**: Explain why changes are made, not just what
- **English only**: All content should be in English for broad accessibility

### Code of Conduct

- Be respectful and constructive
- Focus on the technical merits of contributions
- Help maintain a welcoming environment for all contributors

## Questions?

Open an issue or start a discussion. We're happy to help!
